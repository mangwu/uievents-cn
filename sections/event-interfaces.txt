<h2 id="event-interfaces">基本事件接口</h2>


[[!DOM]] 中定义的基本事件接口是用户界面事件的基础。
这些基本事件接口 <span class="zh-upper">必须</span> 始终在实现中的得到支持: 

*   {{Event}} 接口及其定义在 {{Event}} 
    接口内的常量(constant)、方法(method)和属性(attribute): 
    *   {{Event/NONE}} 常量
    *   {{Event/CAPTURING_PHASE}} 常量
    *   {{Event/AT_TARGET}} 常量
    *   {{Event/BUBBLING_PHASE}} 常量
    *   {{Event/type}} 属性
    *   {{Event/target}} 属性
    *   {{Event/currentTarget}} 属性
    *   {{Event/eventPhase}} 属性
    *   {{Event/bubbles}} 属性
    *   {{Event/cancelable}} 属性
    *   {{Event/composed}} 属性
    *   {{Event/timeStamp}} 属性
    *   {{Event/defaultPrevented}} 属性
    *   {{Event/isTrusted}} 属性
    *   {{Event/stopPropagation()}} 方法
    *   {{Event/stopImmediatePropagation()}} 方法
    *   {{Event/preventDefault()}} 方法
    *   {{Event/initEvent()}} 方法

*   {{CustomEvent}} 接口及其定义在 {{CustomEvent}} 
    接口内的方法(method)和属性(attribute): 
    *	{{CustomEvent/initCustomEvent()}} 方法
    *	{{CustomEvent/detail}} 属性

*   {{EventTarget}} 接口及其定义在 {{EventTarget}} 
    接口内的方法(method): 
    *	{{EventTarget/addEventListener()}} 方法
    *	{{EventTarget/removeEventListener()}} 方法
    *	{{EventTarget/dispatchEvent()}} 方法

*   {{EventListener}} 接口和它的 {{EventListener/handleEvent()}} 方法
*   {{Document}} 接口的 {{Document/createEvent()}} 方法

本规范中定义的事件类型派生自这些基本接口，并且派生自的这些接口的用户事件接口
<span class="zh-upper">必须</span> 继承它们的所有属性、方法和常量。

<h3 id="event-types-list">事件类型列表</h3>

    每个事件 <span class="zh-upper">必须</span> 与一个类型相关联，该类型称为
    <em>事件类型</em> ，并被用作事件对象的 {{Event/type}} 属性。事件类型
    <span class="zh-upper">必须</span> 为 <code>DOMString</code> 类型。

    依据 DOM 支持的级别，或依据用于显示(例如，屏幕)或交互
    (例如，鼠标、键盘、触摸屏或语音)的设备，这些事件类型可以由实现决定和生成。
    当在 [[XML]] 或 [[HTML5]] 应用程序使用(事件类型)时，这些语言的规范 
    <span class="zh-upper">可以</span> 限制与事件类型相关的语义和范围
    (特别是在适合的 <a href="#event-target">事件目标</a> 上)。
    请参阅有关定义所用语言的规范文档，
    以查找这些限制或查找本文档中未定义的事件类型。

    <p class="example">
    以下是对上表进行解释的一种方式: EVENT{load} 
    事件将在该事件的捕获和目标阶段上触发(trigger)附加于 <code>Element</code> 
    节点的 <a href="#event-listener">事件监听器</a> 。此事件不可取消。
    如果 EVENT{load} 事件的 <a href="#event-listener">事件监听器</a> 附加到除了  
    <a>Window</a> ， <code>Document</code> ， 或 <code>Element</code> 
    以外的节点，或者仅附加到冒泡阶段，则不会(在被监听事件目标上)触发此
    <a href="#event-listener">事件监听器</a> 。
    </p>

    <p class="note">
    不要将上表解释为最权威的(definitive)事件类型列表。
    例如， EVENT{load} 事件也用于其他规范中，例如 XMLHttpRequest 中。类似地，
    对于 <strong>任何</strong> 实现了 {{EventTarget}} 接口的对象，
    可以调用对象的 {{EventTarget/dispatchEvent()}} 
    方法将不受信任的事件派发给它的侦听器。
    </p>
